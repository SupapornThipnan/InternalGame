# 🌸 Demetria Spirit — Game Design Document

> เอกสารออกแบบเกม สำหรับนักพัฒนาที่มารับช่วงต่อ
> **โปรเจกต์:** Demetria Spirit (remake) · **ทีม:** indiebear STD (solo dev)
> **Designer:** YamuJung (พี่แยม) · **สถานะ:** HTML prototype → เตรียมพอร์ตเข้า Unity

## 1. เกมนี้คือเกมอะไร

**Demetria Spirit** เป็นเกม **life-sim / farming-sim แนวไอโซเมตริก 2.5D**
ผู้เล่นรับบทเป็นชาวเมืองที่ค่อยๆ ฟื้นฟูดินแดน Demetria ขึ้นมาใหม่
โดยการปลดล็อคพื้นที่ทีละด่าน สร้างห่วงโซ่การผลิต และรับใช้ผู้คนในเมือง

**อารมณ์ที่ต้องการ:** อบอุ่น น่ารัก ผ่อนคลาย ไม่มีการต่อสู้ ไม่มี fail state
ผู้เล่นเดินสำรวจ ปลูกของ คราฟของ ส่งเควส แล้วค่อยๆ เห็นโลกกว้างขึ้นเรื่อยๆ

**เกมอ้างอิงที่ชอบ:** Stardew Valley (loop การทำฟาร์ม), Emil Chronicle Online
(หน้าตา world map แบบ irregular grid), เกมมือถือเกาหลีแนว kawaii (ทิศทางศิลป์)

**แพลตฟอร์มปลายทาง:** Unity C# — ตอนนี้ prototype เป็น HTML เพื่อทดสอบ
ระบบเศรษฐกิจและ feel ก่อนลงทุนเขียน Unity จริง

## 2. Core Gameplay Loop

```
เดินสำรวจแมพ  →  ปลูก/เลี้ยง (passive stage)  →  รอเวลาผ่านไป (วัน)
      ↑                                                    ↓
      │                                              เก็บเกี่ยววัตถุดิบ
      │                                                    ↓
ปลดล็อคด่านใหม่  ←  ใช้ของ + เงิน  ←  ได้เงิน  ←  ส่งเควสให้ NPC
      ↑                                                    ↑
      └──────  คราฟของที่ร้าน NPC  ────────────────────────┘
```

**อธิบายเป็นคำ:**

1. ผู้เล่นเดินไปที่ **passive stage** (ฟาร์ม / คอกสัตว์ / ป่า) กดปลูกหรือเริ่มผลิต
2. เวลาในเกมเดินไป ผ่านไปกี่วันตาม `growDays` ของด่านนั้น
3. กลับมาเก็บเกี่ยว ได้วัตถุดิบ 3 ชนิด อย่างละ `amount` ชิ้น
4. เอาวัตถุดิบไป **craft stage** (ร้าน NPC) เพื่อคราฟเป็นของแปรรูป
5. NPC สุ่มออก **เควส** ขอไอเทม (ทั้งวัตถุดิบดิบและของคราฟ) — ส่งแล้วได้เงิน
6. เอาเงิน + ของ ไป **ปลดล็อคด่านถัดไป** → วนกลับข้อ 1 แต่โลกกว้างขึ้น

**จุดสำคัญของ loop:** เงินมาจากเควสเท่านั้น (ยังไม่มีระบบขายของตรงๆ)
ทำให้ผู้เล่นต้องผลิตของจริงเพื่อจะมีเงิน — ไม่สามารถ grind เงินเปล่าๆ ได้

## 3. ระบบ ECO — หัวใจของเกม

### 3.1 ประเภทของด่าน (Stage)

| ประเภท | `type` | ทำอะไร | ตัวอย่าง |
|--------|--------|--------|---------|
| **Passive** | `passive` | ปลูก/เลี้ยง → รอวัน → เก็บเกี่ยว ได้วัตถุดิบ 3 ชนิด | ฟาร์มพืชผัก, คอกม้า, ป่าไผ่ |
| **Craft NPC** | `craft` | ร้านค้าที่แปรรูปวัตถุดิบเป็นของใหม่ตามสูตร + ออกเควส | ร้านขนมปัง, โรงตีเหล็ก, หอดูดาว |

Passive แบ่งย่อยด้วย `cat` เป็น `farm` / `animal` / `forest` (มีผลกับ filter และธีมภาพ)

### 3.2 จำนวนด่านและ Wave

รวมทั้งหมด **63 ด่าน** แบ่งเป็น **11 Wave** (Wave = ระดับความยาก/ลำดับเนื้อหา)

| Wave | ธีม | จำนวน | ค่าเงินปลดล็อค |
|------|-----|-------|---------------|
| 1 | Starter — ฟาร์ม/ป่า/สัตว์พื้นฐาน + ร้านขนมปัง | 4 | ฟรี (3 ด่านแรก) / 50 |
| 2 | Early — ไร่ผัก ป่าลึก บ่อปลา ร้านซุป โรงไม้ | 6 | 300 |
| 3 | Mid — ข้าวโพด ถ้ำแร่ กระต่าย ผึ้ง กาแฟ BBQ ตีเหล็ก | 7 | 600 |
| 4 | Expansion — ผลไม้ ดอกไม้ หนองน้ำ ชายหาด ม้า ไหม | 8 | 1,200 |
| 5 | Advanced — สมุนไพร ไผ่ นาข้าว พิซซ่า ราเมน เต่า ชา | 7 | 2,000 |
| 6 | Rare — ภูเขาหิมะ ตัดเย็บ ฝ้าย เครื่องประดับ อ้อย | 6 | 3,000 |
| 7 | Endgame — ป่าเรืองแสง เทียน นกแก้ว จระเข้ ศิลปะ หนังสือ | 6 | 5,000 |
| 8 | Special — เห็ด เมล็ดพันธุ์ คลินิกสัตว์ กระบองเพชร ดนตรี | 5 | 8,000 |
| 9 | Legendary — ป่าลึกสุด หอดูดาว | 2 | 12,000 |
| 10 | **South** — ถ่านหิน ซาลาแมนเดอร์ ออบซิเดียน หลอมเหล็ก แก้ว อัญมณีไฟ | 6 | 18,000 |
| 11 | **West** — เหมืองเหล็ก อัญมณี เงิน-ทอง เกราะ เจียระไน กษาปณ์ | 6 | 26,000 |

### 3.3 เงื่อนไขปลดล็อค

แต่ละด่าน (ยกเว้น 3 ด่านแรก) ต้องครบ **3 อย่างพร้อมกัน**:

1. **Prereq** — ด่านก่อนหน้าใน array ต้องปลดล็อคแล้ว (chain เป็นเส้นตรง 100%)
2. **ไอเทม** — `unlockCost` มักมีของคราฟ 2–3 ชิ้นจาก NPC ด่านก่อนหน้า + วัตถุดิบดิบ
3. **เงิน** — `moneyCost` ตาม Wave (ดูตารางข้างบน)

> **ทำไม chain เป็นเส้นตรง:** ตั้งใจให้ผู้เล่นไม่หลง และทำให้ระบบนำทาง
> "ด่านต่อไป" คำนวณได้ง่ายและถูกต้องเสมอ (`stages.find(s => !s.unlocked)`)
> ถ้าจะเปลี่ยนเป็น branching tree ในอนาคต ต้องเขียน logic หา next target ใหม่

### 3.4 Crafting Chain

ของบางอย่างต้องคราฟหลายทอด เช่น:

```
เนื้อกระต่าย + หอม + ขิง  →  ซุปกระดูก
ซุปกระดูก + ข้าวเจ้า      →  ราเมน
```

```
ถ่านหิน + หินแร่  →  เหล็กกล้า  →  ดาบ / โล่ / เกราะเหล็ก
```

ระบบเควสรู้จัก chain นี้ (วน loop จนกว่าจะไม่มีของใหม่เพิ่ม) จึงไม่มีทาง
ออกเควสขอของที่ผู้เล่นยังทำไม่ได้ — **กฎข้อนี้ห้ามพัง**

### 3.5 ระบบเควส

- NPC ที่ปลดล็อคแล้วเท่านั้นที่ออกเควสได้
- เควสสุ่มขอไอเทม 1–3 ชนิด
  - วัตถุดิบดิบ: 2–6 ชิ้น · reward 8–15 เหรียญ/ชิ้น
  - ของคราฟ: 3–8 ชิ้น · reward 15–25 เหรียญ/ชิ้น
- จำกัดสูงสุด 12 ใบพร้อมกัน (ในเกมไอโซ) / 20 ใบ (ใน eco_demo)
- ส่งเควสสำเร็จ → เควสใบใหม่มาเร็วขึ้น

## 4. World Map — แมพเข็มทิศ 5 โซน

### 4.1 แนวคิด

แมพจัดวางตาม **ทิศ** โดยแต่ละทิศมีเอกลักษณ์ทรัพยากรชัดเจน
ผู้เล่นดูแมพครั้งแรกก็รู้ทันทีว่าทิศไหนมีอะไร

```
                    ❄️ เหนือ
                  หิมะ · น้ำแข็ง
                คริสตัล · ป่าลึก
                        │
    ⛏️ ตก ────────  🏰 เมืองหลวง  ──────── 🌾 ออก
  แร่ · เหล็ก        ร้าน NPC ทั้งหมด      ฟาร์ม · สัตว์
  อัญมณี · ทอง       ตลาด · สถานีรถไฟ      ป่า · ชายหาด
                        │
                    🌋 ใต้
                 ลาวา · ถ่านหิน
                  หิน · กำมะถัน
```

### 4.2 การกระจายด่านตามโซน

| โซน | จำนวนด่าน | เนื้อหา |
|-----|----------|--------|
| 🌾 **ออก (east)** | 25 | ฟาร์มทุกชนิด คอกสัตว์ ป่าทั่วไป ชายหาด หนองน้ำ |
| 🏰 **กลาง (center)** | 19 | ร้าน NPC ทั้งหมด (ทุก craft stage อยู่ที่นี่) |
| ⛏️ **ตก (west)** | 8 | ถ้ำแร่ โรงตีเหล็ก + Wave 11 ทั้งหมด |
| 🌋 **ใต้ (south)** | 8 | จระเข้ กระบองเพชร + Wave 10 ทั้งหมด |
| ❄️ **เหนือ (north)** | 3 | ภูเขาหิมะ ป่าเรืองแสง ป่าเรืองแสงลึก |

> ⚠️ **ปัญหาสมดุลที่รู้อยู่แล้ว:** ทิศเหนือมีแค่ 3 ด่าน ดูโล่งกว่าทิศอื่นมาก
> ถ้าจะขยายต่อ แนะนำออกแบบ **Wave 12 — North** (สัตว์หิมะ, คริสตัลน้ำแข็ง,
> ร้านเครื่องดื่มร้อน, ช่างแกะน้ำแข็ง) ให้สมดุลกับทิศอื่น

### 4.3 หลักการวางแมพ (จาก Emil Chronicle Online)

พี่แยมชอบ world map ของ Emil Chronicle เพราะ **ไม่สมมาตร** —
โซนยื่นออกไม่เท่ากัน มีเกาะแยก มีพื้นที่คั่น ทำให้รู้สึกเหมือนทวีปจริง
ไม่ใช่ template ที่ generate มา

ใน prototype ใช้ **value noise** กวนรัศมีของแต่ละโซนให้ขอบหยัก
แล้วมี **ถนน** เชื่อมเมืองหลวงออกไปทุกทิศ

**ตอนพอร์ตเข้า Unity:** ควรเปลี่ยนเป็นแมพที่ออกแบบด้วยมือ (hand-authored tilemap)
แทน procedural — จะคุมองค์ประกอบภาพได้ดีกว่ามาก แต่ยังยึด layout 5 โซนนี้

## 5. ระบบเวลา กลางวัน–กลางคืน

- 1 วันในเกม = 60 วินาทีจริง (ที่ speed 1x) ปรับได้ 1x / 3x / 8x + หยุด
- `time` เป็นเลข 0–1 ต่อวัน: `0` = เที่ยงคืน, `0.5` = เที่ยงวัน
- ช่วงเวลา: กลางคืน → รุ่งเช้า → เช้า → กลางวัน → บ่าย → พลบค่ำ → กลางคืน
- **ผลต่อภาพ:** ท้องฟ้าไล่สี พื้นมืดลง อาคารที่ปลดล็อคแล้วติดไฟหน้าต่าง
  มีแสงกองอุ่นๆ รอบอาคาร ตัวละครถือโคมส่องทาง มี vignette คลุมจอ
- **ผลต่อ gameplay:** ตอนนี้ยังไม่มี — พืชโตตามจำนวน "วัน" ที่ผ่านไปเท่านั้น

> 💡 **ไอเดียต่อยอด:** ให้บาง stage ผลิตได้เฉพาะกลางคืน (เห็ดเรืองแสง, หอดูดาว)
> หรือให้ NPC บางร้านปิดตอนกลางคืน — จะทำให้วงจรเวลามีความหมายกับ gameplay จริง

## 6. ระบบนำทาง (Guidance)

ปัญหาที่เจอตอนเทสต์: ผู้เล่นไม่รู้ว่าต้องไปปลดล็อคด่านไหนต่อ
แก้ด้วยการบอก 4 ชั้นพร้อมกัน:

| ชั้น | รายละเอียด |
|------|-----------|
| 🧭 **เข็มทิศ HUD** | ลูกศรหมุนชี้ทิศจริง + ชื่อด่าน + ระยะ "N ก้าว" · คลิกดูเงื่อนไขได้ |
| 📩 **จดหมาย** | toast แจ้งทันทีหลังปลดล็อคสำเร็จ บอกด่านต่อไป + ทิศ + ของที่ต้องใช้ |
| 💡 **บีคอน** | ลำแสงทองพุ่งขึ้นฟ้าจากด่านเป้าหมาย + ป้ายชื่อลอยตลอด เห็นแต่ไกล |
| 🗺️ **มินิแมพ** | วงแหวนทองเต้นจังหวะรอบจุดด่านเป้าหมาย |

**หลักคิด:** ผู้เล่นไม่ควรต้องเปิด wiki หรือเดา — เกมต้องบอกเองว่าไปไหนต่อ
แต่ยัง "เดินไปเอง" อยู่ (ไม่มี auto-pathing) เพื่อรักษาความรู้สึกสำรวจ

## 7. ทิศทางศิลป์ (Art Direction)

**สไตล์ที่ต้องการ:** 2D isometric แนวเกมมือถือเกาหลี kawaii
- เส้นขอบหนา (thick outline) · cel-shading
- โทนสีอบอุ่น earthy · ไม่ใช่ photorealistic · ไม่ใช่ tilt-shift
- อ้างอิง: ภาพประกอบร้านคาเฟ่สัตว์น่ารัก, ชุด asset ตลาดผลไม้แบบ isometric

**สเปคเทคนิค tilemap (สำหรับ Unity):**

| หัวข้อ | ค่า |
|--------|-----|
| ขนาด sprite ต่อ tile | 256 × 192 px |
| Pixels Per Unit (PPU) | 128 |
| Pivot Y | 0.17 |
| รูปทรง | Isometric diamond |
| ระบบ | Unity Tilemap + **Rule Tile** สำหรับ autotile |

**ขนาดกล่องมาตรฐาน (ProBuilder / prop):**

| ของ | X | Y | Z | หมายเหตุ |
|-----|---|---|---|---------|
| กล่องมาตรฐาน | 1 | 1 | 1 | 1 ช่องกริด |
| Prop ย่อย | 0.25 | 0.25 | 0.25 | sub-grid |
| **แผงรั้ว (fence panel)** | **0.1** | **0.5** | **1.0** | บาง–สูง–ยาว · **ไม่ใช่** 0.25×0.5×0.5 |

> ⚠️ **บทเรียนจากงานจริง:** เคยสั่งงาน "แผงรั้ว" แล้วได้ "เสารั้ว" มาแทน
> เวลาสั่งงาน 3D ให้ระบุ **แกน X/Y/Z เป็นตัวเลขชัดเจน** เสมอ อย่าใช้คำบรรยาย
> และถ้าต้องการชิ้นเดียว ให้ย้ำว่า "1 ชิ้น" — เอา 2 ชิ้นมาต่อกันจะเห็นรอยต่อบนหน้า top

**ระบบ warp:**
- ฟาร์มกลางแจ้ง → อยู่บนแมพหลัก ไม่ต้อง warp (กดปุ่มโต้ตอบตรงนั้นเลย)
- ร้านค้า / โรงเลี้ยงสัตว์ในอาคาร / ป่า → warp เข้าแมพย่อย
- ⏳ **ยังไม่ตัดสินใจ:** คอกสัตว์กลางแจ้ง (ม้า, เต่า, จระเข้, บ่อปลา) จะ warp หรือไม่

## 8. สมดุลเกม (Balance) — สถานะปัจจุบันและปัญหาที่รู้

### ที่ปรับไปแล้ว
- ค่าเงินปลดล็อคคูณ 2 จากตัวเลขตั้งต้น (v0.6.2 ของ eco_demo)
- ของคราฟใน `unlockCost` เพิ่มจาก 1 → 3 ชิ้นต่อด่าน (v0.6.1)

### ⚠️ ปัญหาที่ยังไม่ได้แก้
1. **Wave 1 ฝืดมาก** — มี NPC แค่ร้านเดียว เควสจึงออกช้า เงินเข้าน้อย
   ผู้เล่นต้องรอนานกว่าจะปลดล็อคด่านที่ 5 ได้
2. **ไม่มีระบบขายของ** — เงินมาจากเควสทางเดียว ถ้าเควสสุ่มขอของที่ไม่มี
   ผู้เล่นได้แต่รอ

**ข้อเสนอที่คุยกันไว้ (ยังไม่ทำ):**
- เพิ่มปุ่ม "ขายของ" ที่ตลาด — ราคา 3–5 เหรียญ/ชิ้น (ต่ำกว่าเควสมาก
  เพื่อไม่ให้เป็นทางลัด แต่กันตัน)
- ทำ reward ของเควส scale ตาม Wave ให้ชัดขึ้น

## 9. คำศัพท์ที่ใช้ในทีม

| คำ | ความหมาย |
|----|---------|
| **ด่าน / Stage** | หน่วยพื้นที่ 1 จุดบนแมพ ที่ปลดล็อคและใช้งานได้ |
| **Wave** | กลุ่มด่านตามลำดับความยาก (1–11) |
| **Passive stage** | ด่านที่ปลูก/เลี้ยงแล้วรอเก็บเกี่ยว |
| **Craft stage / NPC** | ร้านค้าที่แปรรูปของและออกเควส |
| **Zone / ทิศ** | โซนบนแมพ: center / north / east / south / west |
| **Prereq** | ด่านก่อนหน้าที่ต้องปลดล็อคก่อน |
| **น้องโค่** | AI ผู้ช่วยออกแบบเกม (Cowork) |
| **น้องเดฟ** | AI นักพัฒนา (Claude Code) — ผู้รับช่วงต่อโปรเจกต์นี้ |
