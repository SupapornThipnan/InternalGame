# 💰 เว็บรายรับ-รายจ่าย — คู่มือพี่แยม

เว็บไฟล์เดียว เปิดจากเครื่องได้เลย ข้อมูลเก็บใน Google Sheet ของพี่แยมเหมือนเดิม
URL เชื่อมต่อ Apps Script ผูกไว้ในตัวเว็บให้แล้ว เปิดมาก็บันทึกข้อมูลได้เลย ไม่ต้องตั้งค่าอะไรเพิ่ม

| ไฟล์ | คืออะไร |
|---|---|
| `index.html` | ตัวเว็บทั้งหมด — ดับเบิลคลิกเปิดได้เลย |
| `apps-script.gs` | โค้ดที่ต้องเอาไปวางใน Google Sheet (ทำให้เว็บบันทึกข้อมูลได้) |
| `README.md` | ไฟล์นี้ |

---

## เปิดใช้ครั้งแรก (ทำ 3 ขั้น ~5 นาที)

### ขั้นที่ 1 — วางโค้ดใน Google Sheet

1. เปิด Google Sheet "รายรับรายจ่าย WEB"
2. เมนูด้านบน → **Extensions** → **Apps Script**
3. ลบโค้ดเดิมที่มีอยู่ให้หมด แล้ววางเนื้อหาทั้งหมดจากไฟล์ `apps-script.gs`
4. กดรูป 💾 (Save)

### ขั้นที่ 2 — สร้างชีตใหม่ + ย้ายข้อมูลเดิม

1. ในหน้า Apps Script เดิม ดูช่องเลือกฟังก์ชันด้านบน → เลือก **`setupSheet`**
2. กด **▶ Run**
3. ครั้งแรก Google จะขออนุญาต → กด **Review permissions** → เลือกบัญชีตัวเอง
   → เจอหน้าเตือนสีเทา ให้กด **Advanced** → **Go to (ชื่อโปรเจกต์) (unsafe)** → **Allow**
   *(ไม่ต้องกลัวครับ มันเตือนเพราะเป็นสคริปต์ที่เราเขียนเอง ยังไม่ได้ส่ง Google ตรวจ)*
4. กลับไปดูใน Sheet จะมีชีตใหม่ชื่อ **"รายการ"** พร้อมข้อมูลเดิม 59 รายการที่ย้ายมาให้แล้ว

> ชีตเดิมยังอยู่ครบ ไม่ได้ลบอะไรเลยครับ

### ขั้นที่ 3 — Deploy

1. ในหน้า Apps Script → กด **Deploy** (มุมขวาบน) → **New deployment**
2. กดรูปเฟือง ⚙️ ข้างคำว่า "Select type" → เลือก **Web app**
3. ตั้งค่า:
   - Execute as: **Me**
   - Who has access: **Anyone**
4. กด **Deploy** → copy **Web app URL** (ลงท้ายด้วย `/exec`)

> URL ที่ deploy ได้ตอนนี้ผูกไว้ในโค้ด `index.html` ให้แล้ว (`DEFAULT_API_URL`) ไม่ต้องเอาไปวางในหน้า
> ⚙️ ตั้งค่าอีก — เว้นแต่ deploy ใหม่แล้วได้ URL คนละอันจริงๆ (ปกติใช้ **Manage deployments → ✏️ Edit
> → Version: New version** จะได้ URL เดิม) หรืออยากลองเชื่อมกับ Apps Script Sheet อื่นแทน
> ก็ใช้ช่องในหน้าตั้งค่าเพื่อ override ได้เหมือนเดิม

เสร็จแล้วครับ 🎉 ทีนี้กดปุ่ม **+** เพิ่มรายการได้เลย ข้อมูลเข้า Sheet ทันที

## วิธีใช้

**📊 Dashboard** — การ์ดสรุป (รายได้ / รายจ่ายรวม / คงเหลือ / จำนวนรายการ),
กราฟวงกลมแยกหมวดหมู่, กราฟแท่งรายรับ-รายจ่ายรายวัน, ตารางสรุปตามหมวด

**📝 รายการ** — ตารางรายการทั้งหมด กดหัวคอลัมน์เพื่อเรียงลำดับ, กดปุ่ม "แก้" เพื่อแก้ไข

**แถบกรองด้านบน** — กรองตาม เดือน / ประเภท / หมวดหมู่ + ช่องค้นหา ใช้ได้ทั้ง 2 หน้า

**ปุ่ม + วงกลมมุมขวาล่าง** — เพิ่มรายการใหม่ ได้จากทุกหน้า

### หมวดหมู่ที่มีให้เลือก
รายได้ · ค่ากิน · ออนไลน์ · ช้อปออนไลน์ · ของใช้ · ของรีวิว · ADS · ลงทุน · สินเชื่อ · จิปาถะ · เดินทาง

### ประเภท
รายได้ · รายจ่าย · จ่ายให้อ้วง · จ่ายสินเชื่อ

## เรื่องที่ควรรู้

- **ตัวเลขตรงกับ Sheet เป๊ะ** — เทสแล้วครับ รายได้ 34,088.52 / รายจ่าย 44,236.82 /
  จ่ายให้อ้วง 4,441 / ADS −2,670 / ค่ากิน 5,779.90 / ช้อปออนไลน์ 10,062.99 /
  ออนไลน์ 32,135.93 / ของใช้ 330 / ของรีวิว 369 — ตรงกับตารางสรุป G/H ทุกช่อง
- **"รายจ่ายรวม" ในเว็บ = ทุกอย่างที่ไม่ใช่รายได้** (รายจ่าย + จ่ายให้อ้วง + จ่ายสินเชื่อ)
  ซึ่งจะไม่เท่ากับช่อง "รายจ่าย" ใน Sheet ที่นับเฉพาะประเภทรายจ่าย
- **เลิกใช้ประเภท "ADS" แล้ว** (รวมเข้ากับ "รายจ่าย") — ถ้ามีข้อมูลเก่าที่ยังเป็นประเภท ADS
  ให้เปิด Apps Script แล้วรันฟังก์ชัน `migrateAdsTypeToExpense` ครั้งเดียว จะแปลงให้เป็น "รายจ่าย" ทั้งหมด
  (หมวดหมู่ ADS ยังใช้ได้เหมือนเดิม แค่ตัดออกจาก "ประเภท" เท่านั้น)
- **URL Apps Script ผูกไว้ในตัวเว็บให้แล้ว** เปิดมาบันทึกได้เลย — ช่องในหน้า ⚙️ ตั้งค่า
  มีไว้เผื่ออยากเปลี่ยนไปเชื่อมกับ Apps Script Sheet อื่นเท่านั้น
- **เว็บไม่ลบข้อมูลได้เลย** — ตั้งใจครับ กันเผลอลบแล้วหายถาวร ถ้าจะลบให้ไปลบใน Sheet เอง
- **ถ้าแก้โค้ด Apps Script ทีหลัง** ต้อง Deploy → **Manage deployments → ✏️ Edit →
  Version: New version** ไม่งั้นเว็บจะยังเรียกโค้ดเวอร์ชั่นเก่า (คนพลาดกันเยอะตรงนี้)
- **เดือนถัดไปไม่ต้องสร้างชีตใหม่** — เว็บใช้ชีต "รายการ" ชีตเดียวยาวๆ แล้วกรองเอาตามเดือน

## ปัญหาที่อาจเจอ

| อาการ | วิธีแก้ |
|---|---|
| ขึ้น "ยังไม่มีชีต รายการ" | ยังไม่ได้รัน `setupSheet` → กลับไปขั้นที่ 2 |
| ขึ้น "เรียก Apps Script ไม่ได้" | URL ผิด — ต้องลงท้าย `/exec` ไม่ใช่ `/dev` |
| กดบันทึกแล้วไม่ขึ้นใน Sheet | Deploy ตอน "Who has access" ไม่ได้เลือก **Anyone** → deploy ใหม่ |
| อ่านข้อมูลไม่ได้เลย | ใน Sheet กด **Share** → ตั้งเป็น **Anyone with the link · Viewer** |
